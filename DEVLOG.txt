IMPLEMENTATION SUMMARY
1. Environment Setup:
   - Initialized the local project folder and installed dependencies through requirements.txt.

2. Model Training Script (app/train_model.py):
   - Loaded the official UCI Cleveland Heart Disease dataset directly from the UCI repository URL to bypass OpenML Sparse ARFF compatibility issues.
   - Cleaned data (handled '?' missing values) and binarized the target variable (0 = absent, 1 = present).
   - Used a scikit-learn Pipeline with ColumnTransformer and StandardScaler for consistent preprocessing.
   - Trained the model with an 80/20 train-test split (stratified).
   - Evaluated model: Accuracy (0.833), Recall (0.786), F1 (0.815), ROC AUC (0.950).
   - Serialized and saved the trained model pipeline using joblib (model.pkl).

3. API Input Schema (app/schemas.py):
   - Implemented strict data validation using Pydantic BaseModel.
   - Defined the 13 clinical input features (age, sex, cp, trestbps, chol, fbs, restecg, thalach, exang, oldpeak, slope, ca, thal) to ensure correct data types from incoming requests.

4. API Server (app/main.py):
   - Built a FastAPI application serving the saved model.
   - Created a root endpoint (/) for health checking.
   - Created a POST endpoint (/predict) that:
     - Accepts JSON payloads matching the Pydantic schema.
     - Converts the payload into a pandas DataFrame.
     - Passes the DataFrame through the scikit-learn pipeline.
     - Returns a JSON response containing the binary prediction, a human-readable risk label, and the calculated probability score.

5. API Testing:
   - Booted the server using Uvicorn.
   - Tested the /predict endpoint using the automatically generated FastAPI Swagger UI (/docs).
   - Successfully received a 200 OK response with prediction outputs.
   - Captured predict_success.png for documentation.
